# Milestone 5: Workspace And Execution Substrate

## Status

Planned only. M5 is not implemented yet.

The repository currently includes the M5 specification so the next stage is
well-defined, but the real MiniAgentOS runtime does not yet provide the
workspace, file editing, or execution substrate described below.

## Objective

Add the first real workspace and execution substrate to MiniAgentOS so the
system can start becoming a genuine coding agent.

M5 is where MiniAgentOS crosses the line from:

"an agent with network/platform tools"

to:

"an agent that can inspect files, modify files, run something, observe the
result, and iterate."

## Why M5 Exists

M4 deliberately stayed honest. It gave MiniAgentOS the right loop shape and the
right tool abstractions, but it did **not** pretend the runtime already had:

- a workspace
- file editing
- executable tasks
- process output observation

Those are exactly the capabilities that make coding agents powerful.

So M5 exists to introduce the minimum substrate needed for the canonical coding
loop:

`inspect -> edit -> execute -> observe -> iterate`

## Product Meaning

By the end of M5, the right mental model should be:

"MiniAgentOS is a bounded coding agent runtime with a real workspace and a real
execution substrate."

This still does not require MiniAgentOS to become a traditional full operating
system or a POSIX clone. It does require the runtime to support the minimum
primitives that make code iteration real rather than simulated.

## Non-goals

- full POSIX compatibility
- unrestricted shell access
- arbitrary package installation
- multi-user permissions
- full process supervision
- persistent general-purpose file system parity with Linux
- large-scale build farm orchestration

## Core Definition

MiniAgentOS M5 is done when the runtime supports:

- one bounded workspace abstraction
- one bounded file editing tool set
- one bounded execution tool
- one observable execution output channel
- one coding-oriented harness suite that validates inspect/edit/run loops

## Required New Capability Areas

### Workspace

MiniAgentOS needs a runtime-owned workspace abstraction. This can be minimal,
but it must let the agent reason about and operate on a controlled file tree.

### File Editing

The agent must be able to inspect and modify files inside that workspace.

### Execution

The agent must be able to launch at least one bounded executable workflow and
observe the result.

This does **not** require a general-purpose shell first. It does require a real
execution primitive with real output.

## Core Loop Requirement

M5 keeps the M4 sessioned tool loop, but extends the tool surface so the loop
can now perform code work.

The important change is not a different loop; it is a more powerful substrate.

## Default M5 Tool Surface

M5 should add at least these tools on top of the M4 set:

- `list_workspace`
- `read_file`
- `write_file`
- `apply_patch`
- `run_process`
- `read_process_output`

Optional later additions:

- `kill_process`
- `list_processes`
- `workspace_search`

### Tool Semantics

- `list_workspace`
  enumerate files and directories within the bounded workspace
- `read_file`
  read one file from the bounded workspace
- `write_file`
  replace or create one file in the bounded workspace
- `apply_patch`
  apply a bounded patch update to one or more files
- `run_process`
  start one bounded execution task
- `read_process_output`
  retrieve output and status for a previously started task

## Honest Execution Requirement

M5 should not hide execution behind fake names like `run_task` if nothing new
is actually happening.

If execution is introduced, it must be real:

- a task starts
- output exists
- the agent can read that output
- the agent can decide what to do next based on it

## Policy Requirement

M5 must keep execution bounded.

At minimum this includes:

- workspace root restriction
- file size or patch size limits
- executable allowlist or runner policy
- output size limits
- timeout limits
- concurrent process limits

## Harness Requirement

M5 requires a new coding-oriented harness suite.

The harness should verify:

- the agent can inspect workspace contents
- the agent can read a file and modify it
- the agent can run a bounded process
- the agent can read process output
- the agent can fix a small failing case through one or more edit/run loops

### Minimum M5 Cases

- `m5-read-file-and-answer`
- `m5-edit-file-and-verify`
- `m5-run-process-and-read-output`
- `m5-fix-small-regression`

## Definition Of Done

M5 is complete when all of the following are true:

- MiniAgentOS keeps the M4 sessioned tool loop
- the runtime exposes a bounded workspace
- the runtime exposes real file read/write/patch tools
- the runtime exposes a real bounded execution primitive
- process output is observable to the loop
- at least one harness case validates a real inspect/edit/run/observe cycle

## Exit Bar

If MiniAgentOS still cannot inspect files, modify files, launch a bounded run,
and observe the output, then it is not yet a coding agent runtime.

If MiniAgentOS can complete a small inspect/edit/run/observe loop inside a
bounded workspace, then the core M5 bar is satisfied.
